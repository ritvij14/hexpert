# Hexpert AI - Tailwind UI Architecture ## 1. Core Principles - **Theme:** Dark mode only. High contrast text on deeply dark backgrounds (`bg-[#09090b]`). - **Typography:** `font-inter` for UI elements and conversational text. `font-geist-mono` for all technical data, addresses, and code snippets. - **Borders:** Thin, subtle borders (`border-zinc-800/80` or `border-zinc-800/60`) used extensively to create structure instead of heavy background changes. - **Accents:** Restrained use of `indigo-500`. Primary buttons are inverted (`bg-zinc-100 text-zinc-950`). ## 2. Common Class Combinations ### Surfaces & Cards - **App Background:** `bg-[#09090b]` - **Card Background:** `bg-[#121214] border border-zinc-800/80 rounded-lg shadow-sm` - **Elevated/User Chat:** `bg-zinc-900 border border-zinc-800/50 rounded-xl shadow-sm` ### Typography Hierarchy - **H1 (Page):** `text-2xl font-semibold tracking-tight text-zinc-100` - **H2 (Card Title):** `text-sm font-medium text-zinc-200` - **Body Text:** `text-sm text-zinc-300 leading-relaxed` - **Secondary Text:** `text-xs text-zinc-400` - **Labels/Meta:** `text-[10px] font-medium uppercase tracking-widest text-zinc-500` ### Interactive Elements (Chips) - **Default Chip:** `px-3 py-1.5 text-xs font-medium rounded-md border border-zinc-700/80 text-zinc-300 hover:bg-zinc-800 transition-colors` - **Active Chip:** `... border-indigo-500/40 bg-indigo-500/10 text-indigo-300 ring-1 ring-indigo-500/20` ### Input Fields - `bg-[#121214] border border-zinc-800/80 rounded-md py-2.5 px-3 text-sm text-zinc-200 placeholder:text-zinc-600 focus:outline-none focus:border-indigo-500/50 focus:ring-1 focus:ring-indigo-500/20` ## 3. Semantic Colors - **Success/Safe:** `emerald-500` (Used for verified badges, positive balances) - **Warning/Med Risk:** `amber-500` (Used for medium audit findings) - **Danger/High Risk:** `red-500` (Used for high severity audit findings) - *Implementation detail:* For badges, use `bg-{color}-500 text-white shadow-{color}-500/20`. For indicators, use `ring-2 ring-{color}-500/20`.

What is EIP-4844? Give me a brief overview and the main benefits.
Explanation

EIP-4844 (Proto-Danksharding) introduces a new transaction type to Ethereum that accepts "blobs" of data. It is a critical stepping stone towards full Danksharding.

Key Benefits

  • Lower L2 Fees Drastically reduces transaction costs on rollups by providing cheaper data availability.
  • Blobspace Data blobs persist on the beacon node for ~18 days before being pruned, avoiding L1 state bloat.
